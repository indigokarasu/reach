# semantic_scholar — academic graph (papers, authors, citations, recommendations)

## What this source has

Semantic Scholar's Graph API exposes ~200M papers across all academic disciplines, with author records, citation/reference graphs, embeddings (SPECTER2), TLDR-style abstract summaries, and a recommendations endpoint trained on the citation graph. Stronger curation than OpenAlex on author disambiguation; weaker breadth.

Paper IDs accept multiple formats: native S2 ID (a 40-char hex), `DOI:10.1038/...`, `CorpusId:12345`, `MAG:1234`, `PMID:12345`, `ARXIV:2401.12345`, `URL:<url>`. Author IDs are S2 author IDs (`1741101`).

Use Semantic Scholar for: paper recommendations from a seed paper, citation context (citing-paper text snippets), author disambiguation. Pair with `openalex` for breadth and pure metadata; pair with `arxiv` for raw preprints.

## Actions

| Action | Purpose | Required params |
|---|---|---|
| `paper_search` | Search papers | `query` |
| `paper` | Single paper by ID (multi-format) | `paper_id` |
| `author` | Author record | `author_id` |
| `recommendations` | Recommended papers given a seed | `paper_id` |

## Worked examples

```bash
# Search for "transformer attention"
python3 scripts/reach.py query semantic_scholar paper_search '{
  "query": "transformer attention",
  "limit": 20,
  "fields": "title,year,authors,abstract,citationCount,influentialCitationCount,tldr"
}'

# Fetch a paper by DOI
python3 scripts/reach.py query semantic_scholar paper '{
  "paper_id": "DOI:10.48550/arXiv.1706.03762",
  "fields": "title,year,authors,abstract,references.title,citations.title"
}'

# Recommendations seeded from a paper
python3 scripts/reach.py query semantic_scholar recommendations '{
  "paper_id": "ARXIV:1706.03762",
  "limit": 10,
  "fields": "title,year,authors,citationCount"
}'

# Author by S2 ID
python3 scripts/reach.py query semantic_scholar author '{
  "author_id": "1741101",
  "fields": "name,affiliations,paperCount,citationCount,hIndex,papers.title"
}'
```

## Response shape

- `paper_search`: `{total, offset, next, data: [{paperId, title, abstract, year, authors: [{authorId, name}], venue, citationCount, influentialCitationCount, openAccessPdf, fieldsOfStudy, tldr: {model, text}, externalIds: {DOI, PubMed, ArXiv, MAG, CorpusId}}, ...]}`.
- `paper`: a single paper object with the same fields, plus optionally `references` and `citations` arrays (each containing nested paper objects, depth limited).
- `author`: `{authorId, name, affiliations, homepage, paperCount, citationCount, hIndex, papers: [...]}`.
- `recommendations`: `{recommendedPapers: [{paperId, title, ...}, ...]}`.

## Pitfalls

- **`fields` parameter is critical.** Without it you get a tiny default subset. Specify what you want using dotted paths: `references.title,authors.affiliations,tldr`.
- **`paper_id` accepts many formats.** Most natural for cross-source work: `DOI:10.xxxx/...`, `ARXIV:1706.03762`, `PMID:12345678`. The bare 40-char hex is the S2 native form. Wrong prefix returns 404.
- **Reference / citation lists are paginated separately.** `references.title` returns up to 1,000; for full citation/reference graph use the dedicated `/paper/{id}/references` and `/citations` endpoints (not currently wrapped here).
- **Rate-limit behavior is harsher than documented.** Anonymous bursts of 5+ req/sec frequently return 429 even within the 100/5min nominal cap. Throttle to ~2/sec.
- **TLDR may be `null`.** Generated by S2's model — only present for indexed papers in supported fields, mostly STEM.
- **`citationCount` vs `influentialCitationCount`.** Influential = papers that cite this work *and* substantively rely on it (S2's model classifies). It's smaller than total citations.
- **Author IDs are imperfect.** Disambiguation has improved but two records for the same person still occur, especially for prolific authors with common names.
- **`fieldsOfStudy` is a small fixed taxonomy** (Computer Science, Medicine, ...) — don't expect detailed sub-fields.
